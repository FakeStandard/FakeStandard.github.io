【.NET】使用 NPOI 匯入 Excel

May 11, 2021 · 2 min · Polar Bear

在網站開發時，總會遇到匯出 EXCEL 的需求，除了匯出之外，偶爾也會遇到匯入 EXCEL 這種懶人需求。

為何是懶人需求呢？通常使用者可以透過 UI 介面來維護資料，包括 CRUD 等功能，所以 Excel 資料會先透過前端介面在將資料傳遞到資料庫儲存。而匯入的方式，可以一次傳遞多筆資料，並且直達資料庫，過程中不需要使用 UI 介面逐筆輸入，非常省時又省力。

註：此篇使用 ASP.NET Core 5 實作

Start#

首先到 NuGet 下載 NPOI 套件並安裝到專案中

建立 ImportController.cs

public class ImportController : Controller
{
    public IActionResult Index()
    {
        return View();
    }
}

建立 Index.cshtml

<h2>Import</h2>

@using (Html.BeginForm("Upload", "Import", FormMethod.Post, new { enctype = "multipart/form-data" }))
{
    <input type="file" class="form-control" name="file" />
    <button type="submit" class="btn btn-primary">Upload</button>
}

在 Controller 中新增一個接收 HttpPost 的 ActionResult Upload，除了返回頁面之外，沒有任何代碼。

/// <summary>
/// 上傳 Excel 方法
/// </summary>
/// <returns></returns>
[HttpPost]
public IActionResult Upload(IFormFile file)
{
    return View("Index");
}

建立以副檔名來判斷要使用的 Workbook 實例私有方法

/// <summary>
/// 判斷副檔名以取得 Workbook 實例，直接讀取檔案內容
/// </summary>
/// <param name="extension"></param>
/// <returns></returns>
private IWorkbook GetWorkbook(string extension, IFormFile file)
{
    MemoryStream ms = new MemoryStream();
    file.CopyTo(ms);

    Stream stream = new MemoryStream(ms.ToArray());

    switch (extension)
    {
        case ".xls":
            return new HSSFWorkbook(stream);
        case ".xlsx":
            return new XSSFWorkbook(stream);
        default:
            return null;
    }
}

回到 Upload 方法撰寫讀取資料程式碼，最後，將讀到的資料作處理或解析，在把整理好的資料存入資料庫，這邊就不撰寫程式碼。

/// <summary>
/// 上傳 Excel 方法
/// </summary>
/// <returns></returns>
[HttpPost]
public IActionResult Upload(IFormFile file)
{
    try
    {
        if (file != null)
        {
            // 檢查 <input type="file" class="form-control" name="file" /> 是否選擇檔案
            if (file.Length > 0)
            {
                // 取得副檔名
                string extension = Path.GetExtension(file.FileName);

                // 取得 Workbook 實例
                IWorkbook wb = GetWorkbook(extension, file);

                if (wb == null)
                {
                    ViewBag.Message = "請上傳 .xls 或 .xlsx 的檔案！";

                    return View("Index");
                }

                // 取得第一個工作表
                ISheet sheet = wb.GetSheetAt(0);

                // 取得標題列
                IRow header = sheet.GetRow(0);

                // 走訪所有資料列(排除標題列)
                for (int row = 1; row <= sheet.LastRowNum; row++)
                {
                    // 驗證不是空白列
                    if (sheet.GetRow(row) != null)
                    {
                        // 走訪所有資料欄
                        for (int column = 0; column < header.LastCellNum; column++)
                        {
                            // 取得資料
                            var data = sheet.GetRow(row).GetCell(column).ToString();

                            // do something...
                        }
                    }
                }

                ViewBag.Message = "上傳成功";
            }
            else
            {
                ViewBag.Message = $"上傳失敗，檔案內無任何資料";
            }
        }
        else
        {
            ViewBag.Message = "請選擇檔案！";
        }
    }
    catch (Exception ex)
    {
        ViewBag.Message = $"上傳失敗，詳細原因:{ex.Message}";
    }

    return View("Index");
}

值得注意的幾個地方

  • 檢查 <input type="file" class="form-control" name="file" /> 是否有選擇檔案
  • 取得索引為零的工作表，建議不直接用名稱取得工作表，除非有特別要求只讀取特定名稱的工作表，否則，優先建議用取得索引的方法來取得 ISheet 實例。
  • 通常範例會包含標題，在讀取時記得要跳過第一行標題
  • 使用 GetRow() 方法取得 Row 之後，要驗證取到的值是否為 Null

Note#

使用第三方套件達成報表匯出匯入功能其實不難，了解整體脈絡後，不管是匯入或匯出，在使用上沒有太大的差別，只是處理取得資料或是返回資料的部分，就會要根據需求來撰寫。
