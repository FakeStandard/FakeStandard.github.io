【Hugo】從頭使用 Hugo 搭配 GitHub Pages

2021-03-23 Hugo Hugo Git GitHub GitHub Pages

Introduce

Hugo 為一款基於 Golang 的靜態網頁生成器，支援 Markdown 語法解析，可快速產生靜態檔案和部署，此部落格也是透過 Hugo + GitHub Pages 打造而成，先前使用 Hexo 在後期打包速度越來越緩慢，耳聞 Hugo 在打包速度上表現非常優異，最後才會選用 Hugo，廢話不多說馬上開始！

Start

對於不同 OS 安裝的方式有所不同，筆者這邊選擇 Windows 裝置使用的套件管理 Chocolatey 來安裝 Hugo。不同 OS 的請參考 Install Hugo

打開 Terminal 安裝 Hugo

$ choco install hugo -confirm

確認 Hugo 版本

$ hugo version

建立新網站，這邊的 sitename 會成為資料夾的名稱

$ hugo new site sitename

Hugo 本身沒有自帶 theme，所以剛開始建置好是沒辦法 run 的，這邊跟著官方使用的 Ananke 主題來套用到新網站，從 github 上拉下一個主題到 site 的資料夾，使用 git submodule 添加到 Themes folder，也可以到 themes 選擇其他主題再解壓到 themes folder 下。

# 移動到專案目錄下
$ cd sitename

# 初始化 git
$ git init

# 添加佈景主題
$ git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

將主題配置到 config.toml

$ echo theme = \"ananke\" >> config.toml

打開 config.toml 查看剛剛配置的 theme 和一些相關設定

# Account of your GitHub
baseURL = "https://username.github.io/"

# 語系
languageCode = "zh-tw"

# Name of your website
title = "How can I help"

# Theme of your website
theme = "ananke"

建立新文章，posts 為 hugo 預設存放文章的路徑，HelloWorld.md 為要創建的檔案名稱和副檔名

$ hugo new posts/HelloWorld.md

使用任何 Editor 開啟剛剛建立的 HelloWorld.md，可以看到一些內容，分別為文章標題、文章建立時間以及設置為草稿，這些設置會被 --- 包圍，包圍的地方稱為 Front Matter，後續會在介紹，現在可以在 --- 下方開始撰寫文章內容！

---
title: "HelloWorld"
date: 2021-03-19
draft: true
---

write something...

在本地啟用 Hugo 服務器，從 http://localhost:1313/ 查看網站，在 hugo server 啟動下，每當編輯或添加新內容，瀏覽器會自動刷新內容，或使用 Ctrl + R 強制刷新。

# -D 為開啟草稿預覽 可以不給
$ hugo server -D

最後通過 hugo 指令就可建立靜態頁面，預設建立的目錄為 ./public/，可以在指令後方添加 -d 或 --destination 來改變存放靜態頁面的路徑，也可在 config 設置 publishdir。

$ hugo

最後，將靜態頁面推送到 GitHub 的 GitHub Pages，這裡只推送 public 資料夾內的靜態頁面，若要推送整個 hugo folder，需另外建立 repository 來儲存。

# 移置 public 目錄下
$ cd public

# 初始化 git
$ git init

# 建立一個遠程倉儲(username 是 github 帳戶)
$ git remote add origin https://github.com/username/username.github.io.git

# 將全部檔案添加到暫存區
$ git add .

# 將暫存區檔案提交出去
$ git commit -m "Initial commit"

# 將本地儲存庫推送到遠程儲存庫
$ git push -u origin master

檢查 GitHub repository 已有推送的最新紀錄，等個幾分鐘再瀏覽 https://username.github.io/ 就大功造成囉！
